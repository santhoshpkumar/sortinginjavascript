Bubble Sort

It is one of the most inefficient sorting algorithm. It compares adjacent items and swaps them if they are in the wrong order. Because of the adjacent comparison and swaps the item bubbles to its correct position and hence the name.

  • O(n^2) average case performance
  • O(n^2) worst case performance
  • O(n) best case performance
  • O(1) auxiliary worst case space complexity

Modified Bubble Sort: Exit the comparison once the entire list is traversed and there were no swaps during that traversal, which indicates that the array is sorted.

Selection Sort

Selection sort breaks the arrays into 2 selections, one on the left that is sorted and one on the right unsorted. To begin with the entire list is unsorted. Pick the left most item and compare it with rest of the item and swap it with the left most if item is in wrong order. The left side of the selections gets sorted as we iterate through the list. We are left with sorted list once all the elements towards the right are exhausted.

  • O(n^2) average case performance
  • O(n^2) worst case performance
  • O(n^2) best case performance
  • O(n) total, O(1) auxiliary worst case space complexity

Insertion Sort

This logic is closet to the way we humans sort. It breaks the list into two section. The left sorted and the right unsorted. Similar to selection sort it picks each untouched element in right and instead of comparing to the rest of items in the right, it places the picked item in the correct spot in the left. To begin with the left will be blank and the right will be the entire unsorted array. Pick each item in right compare against the left and place it in the correct sorted order.

  • O(n^2) average case performance
  • O(n^2) worst case performance
  • O(n) comparison O(1) swap best case performance
  • O(n) total, O(1) auxiliary worst case space complexity

This is efficient when the sort data is small. In fact most of the library (Java, C++, Javascript) switch to use insertion sort for smaller list items, because of its in-place sort capability. It is ineffective for larger list items.

Shell Sort

This is improvement to insertion sort. In insertion sort the gap between the comparison is one, here we perform insertion sort on the sub-list of varying and decreasing gap. When the gap becomes 1 then the array is finally sorted as regular insertion sort. The advantage of using gaps is that the values get distributed so that the small items towards the end gets swapped forward early on.

  • O(n^3/2) | O(n^4/3) average case performance
  • O(n^2) worst case performance
